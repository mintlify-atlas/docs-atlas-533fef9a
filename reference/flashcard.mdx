---
title: "Flashcard"
description: "The Flashcard data model — fields, types, and serialization"
---

A `Flashcard` represents a single question with answer options, correct answer tracking, and study-progress state. It is the core unit stored inside a [Deck](/reference/deck).

## Fields

<ParamField path="card_id" type="string" required>
  Unique identifier for this card. Auto-generated as a UUID v4 on creation. Stable across serialization round-trips.
</ParamField>

<ParamField path="question" type="string" required>
  The question text presented to the learner.
</ParamField>

<ParamField path="options" type="string[]" required>
  Ordered list of answer choices. Each entry is prefixed with a letter and separator, e.g. `"A. London"`, `"B. Paris"`. The prefix letters are used as keys in `correct_answers`.
</ParamField>

<ParamField path="correct_answers" type="string[]" required>
  List of correct option letter keys. Single-answer cards use one entry (`["B"]`); multi-answer cards use multiple (`["A", "C"]`).
</ParamField>

<ParamField path="question_type" type="string" default="single_choice">
  Discriminates between answer modes.

  | Value | Meaning |
  | --- | --- |
  | `single_choice` | Exactly one correct answer |
  | `multiple_choice` | Two or more correct answers |
  | `unknown` | Type could not be determined (e.g. AI extraction fallback) |
</ParamField>

<ParamField path="image_path" type="string | null" default="null">
  Absolute local path to the source image file from which this card was extracted. `null` for manually created cards.

  <Note>Image paths are device-specific. The [Smart Merge sync](/reference/quiz-session) bypasses overwriting this field to prevent cross-device path corruption.</Note>
</ParamField>

<ParamField path="notes" type="string" default="">
  Free-form text notes attached to this card. Cards generated by the AI pipeline receive an automatic warning note indicating they should be reviewed.
</ParamField>

<ParamField path="status" type="integer" default="0">
  Study progress state of this card.

  | Value | Label | Color |
  | --- | --- | --- |
  | `0` | Unseen | Gray |
  | `1` | Learning / Wrong | Orange |
  | `2` | Mastered / Correct | Green |

  <Warning>Changing status semantics is a high-risk operation — both the desktop and mobile clients depend on these integer values.</Warning>
</ParamField>

## Methods

### `to_dict()` → `dict`

Serializes the flashcard to a JSON-serializable dictionary. The `question_type` enum is stored as its string value.

<ResponseField name="card_id" type="string" required>
  UUID string.
</ResponseField>

<ResponseField name="question" type="string" required>
  Question text.
</ResponseField>

<ResponseField name="options" type="string[]" required>
  List of answer option strings.
</ResponseField>

<ResponseField name="correct_answers" type="string[]" required>
  List of correct option letter keys.
</ResponseField>

<ResponseField name="question_type" type="string" required>
  Serialized enum value: `"single_choice"`, `"multiple_choice"`, or `"unknown"`.
</ResponseField>

<ResponseField name="image_path" type="string | null" required>
  Absolute path to the source image, or `null`.
</ResponseField>

<ResponseField name="notes" type="string" required>
  Notes text (may be empty string).
</ResponseField>

<ResponseField name="status" type="integer" required>
  Integer progress state: `0`, `1`, or `2`.
</ResponseField>

---

### `from_dict(data)` → `Flashcard`

Deserializes a `Flashcard` from a dictionary (e.g. parsed JSON). Missing fields fall back to defaults — a new `card_id` UUID is generated if `card_id` is absent.

---

### `get_correct_answer_text()` → `str`

Resolves each entry in `correct_answers` to the full option text from `options`. Supports both letter-key format (`"B"`) and full-text format (the answer is already the full string). Multiple correct answers are joined with ` | `.

Returns `"Unknown"` if no answers can be resolved.

---

### `to_quizlet_row(format_type)` → `str`

Exports this card as a Quizlet-compatible tab-separated row. The `format_type` parameter controls how question and answer are formatted.

| `format_type` | Term side | Definition side |
| --- | --- | --- |
| `"simple"` | Question text only | Full correct answer text |
| `"full"` | `Question >> A \| B \| C` | Full correct answer text |
| `"compact"` | `Question [A \| B \| C]` | Correct answer letter(s) |
| `"safe"` | Uses `{[(CauHoi)]}` / `{[(DapAn)]}` separators for Quizlet custom import | Full correct answer text |

<Tip>Use `"safe"` format when importing into Quizlet with a custom separator configuration to avoid conflicts with tab or newline characters in question text.</Tip>

## JSON example

```json
{
  "card_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "question": "What is the capital of France?",
  "options": [
    "A. London",
    "B. Paris",
    "C. Berlin",
    "D. Madrid"
  ],
  "correct_answers": ["B"],
  "question_type": "single_choice",
  "image_path": "/Users/alex/scans/geography_slide_03.png",
  "notes": "",
  "status": 0
}
```
